PROJECT: Intelligent Analysis of Burn Scar Progression
Date: 2026-02-06

OBJECTIVE
- Provide an overview of the repository, components, technologies, data, model artifacts, runtime details, endpoints, known issues, and developer notes. This file documents the full project structure and behavior to help developers and reviewers understand what's implemented.

SUMMARY
- Purpose: AI system to classify burn degree per-image and analyze progression across time using three model classes:
  - ViT (Vision Transformer): single-image, high-quality analysis
  - CNN (EfficientNet-B0): fast per-image classifier
  - CNN-LSTM (EfficientNet backbone + LSTM + attention): sequence model for progression
- UX: React frontend with pages for Home, Analyze, About, Dashboard, History; cloud-backed history via Firebase.
- Inference: FastAPI backend exposing `/analyze` endpoint supporting single-image and multi-image (sequence) analysis.

BACKEND
Technologies
- Python 3.10+ recommended, FastAPI, uvicorn, PyTorch, torchvision, transformers (for ViT), Pillow, firebase-admin
- Backend entry: `backend/api.py` contains startup, model loaders, `predict_*` helpers and `/analyze` route.

Key endpoints / behavior
- GET `/` : sanity
- GET `/health` : returns model_loaded boolean
- POST `/analyze` : main endpoint supporting three model types
  - Accepts multipart/form-data; tolerant parsing (supports `model`, `model[]`, `model_type`, `files`, `file`, various names)
  - For `vit`: expects single image; runs ViT pipeline (processor + ViTForImageClassification)
  - For `cnn`: single image; runs EfficientNet-B0 based classifier
  - For `cnn_lstm`: expects multiple image files; tries to load CNN-LSTM model; always computes per-image CNN predictions for detailed progression; if cnn_lstm model loads, uses it to compute an overall sequence prediction
  - Returns JSON with:
    - Top-level: `model_used`, `burn_degree`, `burn_degree_index`, `confidence`, `confidence_breakdown`, `healing_stage`, `progression_summary`, `recommendations`, `burn_info`, `imageUrl` (if uploaded)
    - For sequences: `predictions` array (per-image), `trend`, `sequence_prediction` (from cnn_lstm if used), and `progression_report` (full text)

Model loaders
- ViT loader: uses `transformers.ViTImageProcessor.from_pretrained(MODEL_PATH)` and `ViTForImageClassification.from_pretrained(MODEL_PATH)`; expects `model.safetensors` and `config.json` under `Trained Models/vit_burn_model`.
- CNN loader: EfficientNet-B0 from `torchvision.models`, replaces classifier to 3 classes, loads checkpoint from `Trained Models/cnn_model/*.pth`
- CNN-LSTM loader: runtime class implemented in `api.py` to match training: EfficientNet-B0 backbone (classifier replaced with Identity), LSTM (hidden_dim=512, num_layers=2, bidirectional=True), attention layer, classifier. Loader normalizes checkpoint keys (strips `module.`) and tries strict load, then non-strict.

Known backend behaviors & issues
- CNN-LSTM checkpoint mismatch: if saved checkpoint uses different dimension names/shapes or nesting (prefixes like `cnn.`), loading can fail or lead to `strict=False` fallback. When that occurs, backend falls back to per-image CNN predictions and sets `model_used` to `cnn_fallback`.
- Logging: `safe_print()` used to avoid Windows UTF-8 errors, and extensive debug logs are printed for form keys, file counts, model load results.
- Firebase: optional upload to Storage; requires `backend/firebase-service-account.json` or default credentials; code attempts multiple bucket name formats and verifies connectivity.
- Startup: FastAPI app uses an `asynccontextmanager` lifespan that loads ViT on startup. Uvicorn is launched in `if __name__ == '__main__'` with automatic port probing.

- CNN model (cnn_model)

  - Checkpoints: `efficientnet_b0_burn_model_final.pth` (or best)

TROUBLESHOOTING
- If `/analyze` returns `cnn_fallback` and you expect cnn_lstm:
  - Inspect backend logs: they will show strict load errors or size mismatch messages for LSTM weights.
  - Add small debug dump of checkpoint keys and shapes (recommended) to inspect prefixes and dims.
  - Ensure the runtime loader architecture matches training script exactly (hidden_dim, num_layers, bidirectional flag, feature_dim). If training used a different wrapper (e.g., saved `{'model': model.state_dict()}` or nested `cnn.` prefixes), adjust loader accordingly.
- If ViT/transformers load fails:
  - Confirm `Trained Models/vit_burn_model` contains `config.json` and model file.
- Windows console errors: `safe_print` helps avoid UnicodeEncodeError; ensure locale/terminal supports UTF-8.
- Port bind errors: `api.py` probes ports 8000-8004 automatically.
